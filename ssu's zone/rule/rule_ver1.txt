[ 서비스 개요 ]
----------------------------------------------------------------
- 사용자가 좋아하는 등번호(0~99)를 입력
- KBO 10개 구단에서 해당 번호를 단 선수 조회
- 급조팬지수 기반 팀 순위 1~3위 반환
- 팬 타입 분류 결과 함께 제공


[ 등번호 범위 ]
----------------------------------------------------------------
- 최소: 0번
- 최대: 99번
- 범위 초과 시 422 오류 반환


[ 급조팬지수 계산 (0~100점) ]
----------------------------------------------------------------
팀별 선수 중 가장 높은 점수를 해당 팀의 대표 점수로 사용.

■ 1순위 — WAR 기반 (승리기여도 직접 지표)
  WAR 3.0 이상  → 88점
  WAR 2.0 이상  → 74점
  WAR 1.0 이상  → 58점
  WAR 0.5 이상  → 42점
  WAR 0.5 미만  → 28점

■ 2순위 — OPS/ERA 기반 (carry 선수, WAR 없을 때)
  타자 OPS:
    0.950 이상  → 78점
    0.900 이상  → 66점
    0.800 이상  → 52점
    0.700 이상  → 36점
    0.700 미만  → 24점

  투수 ERA:
    2.00 이하  → 78점
    2.80 이하  → 66점
    3.80 이하  → 52점
    4.80 이하  → 36점
    4.80 초과  → 24점

■ 3순위 — 비carry 1군 선수
  15점 고정 (동점 허용)

■ 최근 폼 보정 (-8 ~ +10점)
  타자 기준 (최근 5경기 평균 타율):
    0.380 이상  → +10점
    0.300 이상  →  +5점
    0.200 이상  →   0점
    0.200 미만  →  -8점

  투수 기준 (최근 5경기 평균 ERA):
    1.50 이하  → +10점
    3.00 이하  →  +5점
    5.00 이하  →   0점
    5.00 초과  →  -8점

■ 최종 점수
  base + 폼 보정, 최대 100점 상한


[ carry 선수 판정 기준 ]
----------------------------------------------------------------
KBO 공식 시즌 랭킹 기준:
  - 타자: OPS TOP 30 이내
  - 투수: ERA TOP 25 이내
해당 범위 내 선수 → is_carry = True
그 외 1군 등록 선수 → is_carry = False


[ 팀 추천 결과 구성 ]
----------------------------------------------------------------
- 상위 3팀 반환 (팀 수가 3개 미만이면 있는 만큼만)
- 카드 배치: 2위 | 1위 | 3위 순서

■ 급조팬지수 설명 문구
  80점 이상  → "이 번호와의 인연은 거의 운명 수준입니다."
  60점 이상  → "이 번호 선수가 활약 중입니다."
  60점 미만  → "강한 인연은 아니지만 문은 열려있습니다."

■ no_carry 처리
  해당 번호 전체 선수 중 carry가 한 명도 없으면
  → noCarryBanner 표시 ("어떤 팀이든 좋아요" 분기)

■ 선수 없음 처리 (geongseongpa)
  해당 번호 선수가 0명이면
  → geongseongpaBanner 표시


[ 팬 타입 분류 ]
----------------------------------------------------------------
■ 전체 선수 분포 기반 (해당 번호 모든 구단 선수 대상)

  투수콤: 전체 중 투수 비율 50% 이상
  타자콤: 전체 중 비투수 비율 50% 이상 (투수콤 미해당 시)

  내야콤: 내야수(1루/2루/3루/유격/내야) 4명 이상 + 전체 35% 이상
  외야콤: 외야수(좌/중/우/외야) 4명 이상 + 전체 35% 이상
          (내야콤과 외야콤은 둘 중 하나만, 내야콤 우선)

■ 1위 팀 선수 기반 (1위 팀의 대표 선수 기준)

  포수콤: 1위 선수 포지션이 포수
  유격콤: 1위 선수 포지션이 유격수
          (포수콤, 유격콤은 둘 중 하나만)

  얼라콤: 1위 선수 나이 25세 이하
  거포콤: 1위 선수 SLG 0.500 이상

■ 팬 타입 설명 문구
  투수콤: "마운드 위에서 분투하는 선수들의 모습이 보입니다."
  타자콤: "공을 친 그들의 방망이가 외치는 승리의 함성이 들립니다."
  내야콤: "옷에 묻어있는 흙이 그들의 열정을 보여줍니다."
  외야콤: "가장 외로운 곳에서 가장 위험한 순간을 막는 선수들의 몸을 던지는 플레이가 보입니다."
  포수콤: "가장 낮은 곳에서 가장 무거운 책임을 가진 선수의 뒷모습이 보입니다."
  유격콤: "가장 많은 타구와 경쟁하는 흙이 묻은 유니폼의 펄럭거림이 보입니다."
  얼라콤: "어떤 새싹은 그 누구보다 굳건한 거목의 잠재력을 가지고 있습니다."
  거포콤: "그 선수의 등장으로 팬들의 마음도 경기의 흐름도 뒤바꾸는 모습이 보입니다."

  팬 타입 1개: 해당 타입 설명 문구 출력
  팬 타입 복수: "활약하는 선수들이 다양한 길로 향하네요!"
  팬 타입 없음: "이 번호를 달고 있는 선수들의 특성을 분석했습니다."


[ 데이터 구조 ]
----------------------------------------------------------------
■ data/roster.json
  - 갱신 주기: 매주 월요일 오전 08:00 (KST) 자동
                트레이드/외국인 영입 시 GitHub Actions 수동 실행
  - 포함 정보: 팀명, 선수명, 등번호, 포지션, 투타, 생년월일, 나이

■ data/stats.json
  - 갱신 주기: 매일 오전 03:00 (KST) 자동
  - 포함 정보: OPS/ERA TOP 랭킹 (carry 판정), WAR TOP 10

■ 데이터 출처
  - 로스터: KBO 공식 (koreabaseball.com/Player/RegisterAll.aspx)
  - 스탯:   KBO 공식 타자/투수 시즌 랭킹
  - WAR:    statiz.co.kr 메인 TOP 10


[ API ]
----------------------------------------------------------------
GET /api/recommend?number={0~99}
  → 팀 순위 + 팬 타입 JSON 반환

GET /api/health
  → 서버 상태 + 데이터 갱신일 반환
