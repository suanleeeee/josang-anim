3. 서비스 흐름
----------------------------------------------------------------

[인트로 화면]
  좋아하는 숫자 입력 (0 ~ 99)
       ↓
[API 호출] GET /api/recommend?number=N
       ↓
[백엔드 처리]
  1. KBO 10개 구단에서 해당 등번호 선수 조회
  2. is_carry 판정
     - 타자: KBO 공식 OPS TOP30 포함 여부
     - 투수: KBO 공식 ERA TOP21 포함 여부
  3. 팀별 급조팬지수 계산 (상위 3팀 선정)
  4. 팬 타입 분류 (입덕 가이드)
       ↓
[결과 화면]
  포디움 카드 (2위 | 1위 | 3위 순서 배치)
  입덕 가이드 (팬 타입 배지 + 설명)
  1위 팀 추천 이유


----------------------------------------------------------------
4. 급조팬지수 공식 (불펜 설계)
----------------------------------------------------------------

승리기여도 중심 3단계:

  1순위: WAR 기준
     WAR ≥ 3.0  →  88점
     WAR ≥ 2.0  →  74점
     WAR ≥ 1.0  →  58점
     WAR ≥ 0.5  →  42점
     WAR < 0.5  →  28점

  2순위: WAR 없을 때 OPS/ERA 기준
     OPS ≥ .950 또는 ERA ≤ 2.00  →  78점
     OPS ≥ .900 또는 ERA ≤ 2.80  →  66점
     OPS ≥ .800 또는 ERA ≤ 3.80  →  52점
     OPS ≥ .700 또는 ERA ≤ 4.80  →  36점
     그 외                        →  24점

  3순위: 스탯 없는 1군 선수  →  26점 (기본값)

  최근 폼 보정: -8점 ~ +10점 추가 적용
  팀 대표 점수: 팀 내 가장 높은 점수 선수 기준


----------------------------------------------------------------
5. 결과 분기 처리
----------------------------------------------------------------

일반 케이스 (carry 선수 있음)
  → "당신의 운명 구단" + [팀명]의 팬이 될 가능성이 가장 높습니다
  → 포디움 카드 + 입덕 가이드 + 1위 팀 추천 이유

no_carry (carry 선수 없음)
  → "자유로운 번호" + 이 번호를 단 선수라면 어떤 팀이든 좋아요
  → 포디움 카드 + 입덕 가이드 (추천 이유 숨김)
  → 포지션 기반 팬 타입(포수/유격수/유망주/거포) 생략

개성파 (해당 등번호 데이터 없음)
  → "무한한 가능성을 품은 당신!" 배너
  → 입덕 가이드만 표시

동점 처리
  → random.shuffle 후 정렬 → 특정 팀이 항상 1위 되는 현상 방지


----------------------------------------------------------------
6. 입덕 가이드 (팬 타입 분류)
----------------------------------------------------------------

전체 선수 분포 기반:
  투수    — 전체 중 투수 비율 50% 이상
  타자    — 전체 중 비투수 비율 50% 이상
  내야수  — 내야수 4명 이상 + 35% 이상
  외야수  — 외야수 4명 이상 + 35% 이상

1위 선수 기준:
  포수    — 1위 선수 포지션이 포수
  유격수  — 1위 선수 포지션이 유격수
  유망주  — 1위 선수 만 23세 이하
  거포    — 1위 선수 장타율(SLG) .500 이상

8. 주요 화면 구성
----------------------------------------------------------------

인트로 화면
  배경: 야간 경기장 SVG (조명탑/잔디/별)
  입력: 숫자 0~99 / 운명 확인하기 버튼
  정확도 안내: 0~50 높음 / 51~80 보통 / 81~99 낮을 수 있음

결과 화면
  배경: 응원 열기 SVG (컨페티/응원봉/야구공, 노란 계열)
  상단 28vh: 여백 (배경 노출)
  구단명 헤더 → 포디움 카드 → 입덕 가이드 → 추천이유
  하단: 이미지 저장 / 링크 복사 / 다시하기 / about 버튼

공유 카드 (이미지 저장 시 별도 생성)
  노란 그라데이션 배경 (#FFF4CC → #FFE8A0 → #FFDCA0)
  팀명 헤더 + 입덕 가이드 카드 + 1위 팀 추천이유 카드
  경고문: 실제 선수 추천·투자 등과 무관한 재미용 서비스입니다.

About 모달 (2페이지)
  1페이지: 기획의도 (라야 작성)
  2페이지: AI 팀원 후기 6명 + 라야 후기

10. 주의사항 및 알아두면 좋은 것들
----------------------------------------------------------------

- Render는 Python 버전을 자동으로 올릴 수 있음
  → .python-version 파일로 버전 고정 필수

- 배포 후 반영까지 2~3분 소요
  브라우저 캐시 문제 시 Ctrl+Shift+R (강제 새로고침)

- 구단 설명 문구(TEAM_VIBES)는 이번 버전 미사용

- 삼성 팬이라고 삼성에 좋은 조건 달지 않았습니다 (라야 확인)
